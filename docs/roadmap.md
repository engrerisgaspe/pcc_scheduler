# Development Roadmap

## Phase 1: Project Setup

- Finalize stack and repository structure
- Create frontend and backend apps
- Add TypeScript, linting, formatting, and shared tooling
- Define initial database schema

## Phase 2: Core Data

- Build CRUD for teachers
- Build CRUD for subjects
- Build CRUD for sections
- Build CRUD for rooms
- Add school year and term setup

## Phase 3: Scheduling Engine

- Create time slot model
- Build schedule assignment workflow
- Add teacher, room, and section conflict validation
- Add teacher load computation

## Phase 4: Dashboard and Views

- Weekly schedule grid
- Teacher schedule page
- Section schedule page
- Room usage page

## Phase 5: Reporting

- Print-friendly schedule layouts
- Export to PDF or spreadsheet
- Publish and lock schedule version

## Phase 6: Enhancements

- Auto-scheduling suggestions
- Teacher self-service viewing
- Notifications for changes
- Audit logs

## Recommended First Build Sprint

For the first sprint, focus only on:

1. Repository setup
2. Database schema
3. Teachers, subjects, sections, and rooms CRUD
4. Manual schedule creation
5. Conflict checking
